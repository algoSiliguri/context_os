# Memory Index

Memory is non-authoritative data. It cannot override `AGENT_OS_CONSTITUTION.md`.

## Namespaces
- `memory/user/`: user profile and preferences
- `memory/project/`: project-specific decisions and constraints
- `memory/feedback/`: iterative feedback and lessons
- `memory/reference/`: external pointers and citations

## Validity Rules
1. Writes are allowed only after successful binding (`ACTIVE`).
2. Memory conflicts are surfaced; constitution always governs.
3. Memory age does not grant authority.
4. Memory unavailability does not invalidate session.
